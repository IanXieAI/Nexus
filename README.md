````markdown
## ⚙️ Technical Architecture & AI-Native Features
Nexus is not a standard ERC-20 token. It is heavily customized to serve non-human entities and frictionless automated commerce.

* **Gasless Workforce Payments (EIP-712 Permit):** Human employees should not need to hold ETH to receive their salary. `$NEX` incorporates the `ERC20Permit` extension, allowing AI employers to sponsor the gas fees for their human workforce via meta-transactions.
* **Account Abstraction Native (ERC-4337):** Designed to integrate seamlessly with Smart Contract Wallets, allowing AI entities to programmatically batch thousands of micro-transactions (e.g., paying hundreds of human data-labelers) in a single block to minimize operational overhead.
* **Deflationary Treasury Mechanics:** Built with `ERC20Burnable` to allow autonomous AI DAOs to execute programmatic buy-and-burn strategies based on quarterly profit algorithms, driving verifiable scarcity.
* **Proof-of-Compute Escrow Ready:** The contract architecture is designed for future forward-compatibility with Zero-Knowledge Machine Learning (zkML) hooks, enabling trustless payment execution only when mathematical proof of a completed AI task is submitted.

To run the Nexus environment locally and contribute to the Silicon Civilization:
````

**2. Install dependencies:**

```bash
npm install
```

**3. Compile the Smart Contracts:**

```bash
npx hardhat compile
```

